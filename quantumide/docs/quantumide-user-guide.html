03 / Documentation route

Agent Sessions

Deep link ready

Overview

Use ChatGPT/OpenAI agent sessions, customization, synced prompts, and remote agent hosts. For local OpenAI development, copy .env.example to .env and set QUANTUMIDE_OPENAI_API_KEY. Optional overrides are QUANTUMIDE_OPENAI_BASE_URL and QUANTUMIDE_OPENAI_MODEL.

Live activity steps

During a turn, chat shows tool cards (search, read file, propose edit) as the agent works—not only the final answer. The session list shows the current step (e.g. Read app.ts). Streamed answers and activity steps appear together. Cancel keeps partial work and marks in-flight tools as cancelled.

Smart tip

When the chat panel says Analyzing or Thinking, the Agent Host is preparing context or waiting on the model. Debug timeline: View → Output → QuantumIDE Agent. Toggle steps with Settings → QuantumIDE AI → Show activity steps.

Checklist

  • Confirm .env contains QUANTUMIDE_OPENAI_API_KEY.
  • Launch with npm run dev or the packaged app.
  • Open Chat and verify ChatGPT / OpenAI is selected.
  • Send a prompt that needs workspace search/read and confirm tool cards appear before the answer.
  • Optional: open Output → QuantumIDE Agent to see the step timeline.

Operations

Command center

# Local OpenAI setup
cp .env.example .env
# Edit .env and set QUANTUMIDE_OPENAI_API_KEY. Optional: QUANTUMIDE_OPENAI_BASE_URL, QUANTUMIDE_OPENAI_MODEL.

# Development launch: clean, compile, and launch dev workbench with Node 22
cd "/Users/abditeramu/Documents/repos/StillnessCompiler/quantumide" && ./scripts/ensure-node22.sh bash -lc 'rm -rf out node_modules/.cache && npm run compile && ./scripts/code.sh'

# Production build: clean and package (macOS arm64)
cd "/Users/abditeramu/Documents/repos/StillnessCompiler/quantumide" && ./scripts/ensure-node22.sh bash -lc 'set -e; cp product.json product.json.quantumide-build.bak; trap "mv product.json.quantumide-build.bak product.json" EXIT; python3 - <<'"'"'PY'"'"'
import json
from pathlib import Path
p = Path("product.json")
data = json.loads(p.read_text())
data["builtInExtensions"] = []
data["webBuiltInExtensions"] = []
p.write_text(json.dumps(data, indent=2) + "\n")
PY
rm -rf out out-build out-vscode out-vscode-min .build ../VSCode-darwin-arm64 node_modules/.cache && npm run compile && npm run gulp vscode-darwin-arm64'

# Production install and launch (run from quantumide after build)
cd "/Users/abditeramu/Documents/repos/StillnessCompiler/quantumide"
rm -rf "/Applications/QuantumIDE.app"
ditto "../VSCode-darwin-arm64/QuantumIDE.app" "/Applications/QuantumIDE.app"
env -u ELECTRON_RUN_AS_NODE open "/Applications/QuantumIDE.app"

Production rebuild (required for code fixes)

Do not rely on npm run compile alone — that only updates out/ for dev. The .app is built from gulp vscode-darwin-arm64 (esbuild bundle). Use:

cd quantumide && ./scripts/rebuild-quantumide-mac.sh

Uses npm run gulp (not bare gulp) and clears builtInExtensions during the build so packaging does not fail on GitHub extension downloads.

Browser console (DevTools)

After a correct production rebuild, DevTools open automatically in a detached window shortly after launch. Shortcuts: F12, Cmd+Option+I (mac). Launch flag: open -a QuantumIDE --args --open-devtools. Disable auto-open: QUANTUMIDE_NO_AUTO_DEVTOOLS=1.
